# Contributing to DoneBond

Contributions are welcome. Please open an issue first to discuss the change.

## Development expectations

1. Read the relevant specification and `task.md`.
2. Branch from the current integration branch and keep changes focused.
3. Never commit credentials, evidence containing secrets, wallet material, or
   production environment values.
4. Add or update tests for behavioral changes.
5. Run the relevant root quality gates before requesting review.
6. Use Conventional Commit messages and explain any security or architecture
   tradeoffs in the pull request.

Security vulnerabilities should not be opened as public issues. Until a private
reporting channel is published, contact the repository owner privately through
their GitHub profile and include only the minimum reproduction details needed.
